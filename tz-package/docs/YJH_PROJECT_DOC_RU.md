# YJH Your Job's Here: Документация проекта

## 1. О проекте

`YJH Your Job's Here` — QA-платформа для управления качеством, объединяющая:
- баг-трекинг,
- тест-менеджмент,
- wiki-базу знаний,
- канбан-доску,
- аналитику и дашборды.

Цель: заменить связку разрозненных инструментов (Jira/TestRail/Confluence) единым решением.

## 2. Текущая запись проекта в БД

- Таблица: `projects`
- Название: `YJH Your Job's Here`
- Статус: `active`

## 3. Технологии

- Backend: PHP 8+ (страницы + API)
- DB: SQLite (`database.sqlite`)
- Frontend: Bootstrap 5, Chart.js, JS
- Безопасность: CSRF, подготовленные SQL-запросы (PDO), экранирование вывода

## 4. Ключевой функционал

### 4.1 Баги
- Создание/редактирование багов
- Комментарии и история
- Фильтрация и приоритеты
- Привязка к проектам

### 4.2 Тесты
- Планы, наборы, кейсы, тест-раны
- Исполнение тестов
- Статусы pass/fail/blocked/not_tested

### 4.3 Канбан
- Колонки и карточки
- Drag & Drop
- Редактирование/удаление карточек
- Редактирование/удаление колонок
- Проверка на пустоту при удалении колонки

### 4.4 Wiki
- Создание/редактирование страниц
- История версий
- Сравнение версий

### 4.5 Дашборд и отчеты
- Pass/Fail, High Priority Bugs, Bug Trends
- Risk Engine с формулой и подсказкой
- Кликабельные графики (drilldown)

## 5. Что уже внесено в рамках текущего цикла

- Исправлены критические ошибки (`bug.php`, `project.php`, CSRF в Wiki)
- Улучшена темная тема (контраст, таблицы, формы, календарь, compare-блок)
- Исправлено поведение переключения языка
- Исправлен `Ctrl+K` быстрый поиск
- Добавлен CRUD для проектов (create/edit/delete)
- Улучшен интерфейс пользователей (вкладки New User / Users Base)
- Устранены warning при redirect после вывода через безопасный fallback

## 6. Баг-репорты проекта

В таблицу `bugs` добавлены записи по согласованному листу:
- всего: `22` записи
- исправленные: `21` (`status=closed`)
- идея: `1` (`status=new`)

## 8. Локальный запуск

1. Открыть проект в `XAMPP` (или `php -S`).
2. Убедиться, что доступен файл `database.sqlite`.
3. Открыть в браузере:
- `/index.php`
- `/bugs.php`
- `/projects.php`
- `/admin/users.php`

## 9. Проверка после обновлений

1. Создать/обновить/удалить проект в `projects.php`.
2. Проверить вкладки пользователей в `admin/users.php`.
3. Проверить темную тему на страницах таблиц и форм.
4. Проверить `Ctrl+K` и открытие глобального поиска.
5. Проверить баг-лист проекта по фильтру проекта.
